# Phase 1: Scaffold & Robots.txt Gate

**Goal:** Create a clean project skeleton and a `robots.txt` checker that acts as the first gate before any scraping occurs. This ensures compliance with site rules from day one.

> **Learning Objective:** Project structuring, working with `robots.txt` in Python, async fetching, and basic logging setup.

## What Gets Built
A clean project skeleton plus a robots checker. If an allowed site blocks us, we flag it in a local JSON file for later manual review rather than skipping it silently.

## Deliverables

### 1. `page_search/config/brands.json`
The master list of brands the scraper will process. Each entry holds its size chart URL, chart type, and optional per-brand overrides.

```json
[
  {
    "brand_name": "Uniqlo",
    "base_url": "https://www.uniqlo.com",
    "product_url": "https://www.uniqlo.com/in/en/products/E485454-000/00",
    "chart_type": "html",
    "region": "IN",
    "high_risk": false
  }
]
```

### 2. `core/robots.py`
Fetches and parses `robots.txt` for any given domain.
- Uses `urllib.robotparser.RobotFileParser` or an async equivalent.
- Checks if the specific size chart URL path is disallowed.
- Extracts `Crawl-delay` if specified to respect politeness rules.

### 3. `outputs/blocked.json`
Auto-generated by `core/robots.py`. Brands that block scraping are written here to be flagged for manual entry.

### 4. `core/logger.py`
Consistent logging setup using Python's built-in `logging` module. Console output initially to verify execution.

### 5. `page_search/run.py`
The entry point for Stage 2. For Phase 1, it only:
1. Loads `page_search/config/brands.json`.
2. Runs the robots check for each brand.
3. Prints `allowed` or logs as `blocked`.
4. Exits cleanly.

## Acceptance Criteria
- [ ] Running `python -m page_search.run` checks `robots.txt` and logs allowed/blocked status.
- [ ] Blocked brands are successfully recorded in `outputs/blocked.json`.
- [ ] Crawl delays are logged and stored if specified by the site.
